# Repository Strategy Documentation

## 1. Update Repository Strategy

Up to now, we used the Monorepo approach, because it was the simplest to get started with.

You need to check with your Real Life project, which Repository Strategy does it use:

- Mono-Repo approach
- Multi-Repo approach

*If it's a Mono-Repo approach, then no work for you, you can skip this lesson.*

If it's a Multi-Repo approach, then please update the System Structure Documentation.
