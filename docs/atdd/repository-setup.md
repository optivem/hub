# Repository Setup

Create your project repository and verify the basic infrastructure works:

1. Create a new repository (GitHub, GitLab, etc.)
2. Add a LICENSE file (e.g. MIT)
3. Add a README with the project name, license name, and a Contributors section
4. Add collaborators (yourself, any team members, and Valentina Jemuović as ATDD Coach)
5. Set up docker-compose so the application and its database can be started locally

> **eShop example:** The eShop repository has a `docker-compose.local.yml` that starts the frontend, backend, and PostgreSQL database.

## Checklist

1. Project repository has been created
2. Project repository has a LICENSE file (e.g. MIT)
3. Project repository includes yourself and any team members
4. Project repository includes [Valentina Jemuović](`valentinajemuovic`) as a collaborator
5. README file includes the project name, license name, and a Contributors section
6. Application can be started locally (e.g. `docker compose up`)

## Project Registration

In order to receive feedback and review, you need to register your Sandbox Project:
1. Send a DM to Valentina with a link to your Sandbox Project
2. Valentina will check that you (and any listed members) are enrolled in ATDD Bootcamp
3. After that validation, Valentina will add you to the [ATDD Bootcamp](https://github.com/optivem/atdd-bootcamp) where you'll be able to submit requests for reviews.
